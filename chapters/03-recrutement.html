Un dev a besoin des infos classiques mais parfois oubliées: quelle est la vision de du business, quelles valeurs Dans quel cadre est-ce qu'il va évoluer, quelle ambiance, méthode de travail Qu'est-ce qu'on va attendre du développeur ? Présenter les pratiques de dev et projets, les choix techno principaux, le fonctionnement global de l'entreprise pour le developement logiciel Est-ce que le dev partage ces pratiques de dev ? ces technos l'intéressent ? Est-ce que le dev est dans le même mindset que l'équipe. Agile OK ? TDD OK ? Code review OK ? Craftsmanship OK ? Un QCM peut permettre de valider techniquement avec un petit peu de pair programming c'est top !
